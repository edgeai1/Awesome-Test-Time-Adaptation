## 一、研究背景

CLIP通过大规模图文对比预训练，实现了强大的零样本分类能力：给定一张图片和一组文本提示（如"a photo of a [class]"），CLIP计算图像嵌入与每个类别文本嵌入的余弦相似度来进行分类。

然而，零样本能力高度依赖于**文本提示的设计**。后续工作CoOp提出在**有标注数据**上学习连续提示向量，显著提升下游任务性能，但需要目标域标注。CoCoOp引入图像条件化提示，仍需标注数据。

**核心矛盾**：提示调优能显著提升性能，但需要标注数据；零样本场景下恰恰没有标注数据。

## 二、动机 (Motivation)

TPT提出了一个优雅的解决方案：**在测试时为每个测试样本自适应地优化提示，不需要任何标注数据**。

核心直觉：对于一个好的提示，模型应该对同一张图片的不同增强版本给出**一致的预测**。如果增强后的预测不一致（高熵），说明当前提示不够好。因此，可以通过**最小化多个增强视图的平均预测熵**来优化提示。

但并非所有增强视图都有意义——一些激进增强可能改变图像语义。因此需要**置信度选择**：只保留高置信度（低熵）增强视图参与优化。

## 三、方法详解

### 3.1 问题形式化

给定CLIP模型（图像编码器 $f_I$、文本编码器 $f_T$），C个类别名 $\{c_1, ..., c_C\}$，可学习提示向量 $\mathbf{p} = [v_1, v_2, ..., v_M]$。

类别 $c_k$ 的文本输入为 $[v_1, ..., v_M, c_k]$，文本嵌入 $\mathbf{t}_k = f_T([v_1, ..., v_M, c_k])$。

图像 $x$ 的嵌入 $\mathbf{z} = f_I(x)$。

预测概率：

$$p(y=k|x) = \frac{\exp(\text{sim}(\mathbf{z}, \mathbf{t}_k) / \tau)}{\sum_{j=1}^C \exp(\text{sim}(\mathbf{z}, \mathbf{t}_j) / \tau)}$$

### 3.2 TPT核心流程

对每个测试样本 $x$：

**Step 1 — 增强**：生成 N=63 个增强视图 $\{x^{(1)}, ..., x^{(N)}\}$（AugMix风格：随机裁剪+颜色抖动）。

**Step 2 — 置信度选择**：对每个增强视图计算预测熵 $H^{(i)}$，仅保留Top 10%低熵视图集合 $\mathcal{S}$。

**Step 3 — 熵最小化**：对保留视图计算平均预测：

$$\bar{p}(y=k) = \frac{1}{|\mathcal{S}|} \sum_{i \in \mathcal{S}} p(y=k|x^{(i)})$$

优化目标：$\mathcal{L} = H(\bar{p}) = -\sum_k \bar{p}(y=k) \log \bar{p}(y=k)$

通过1步梯度下降更新提示向量 $\mathbf{p}$。

**Step 4 — 预测与重置**：用更新后的提示对原始图像预测，然后**重置提示**回初始值。

### 3.3 流程图

```
输入图像 x
    │
    ├──→ 生成63个增强视图 {x¹, x², ..., x⁶³}
    │         │
    │    CLIP图像编码器 → {z¹, z², ..., z⁶³}
    │         │
    │    计算每个视图预测熵 → 保留Top 10%低熵视图
    │         │
    │    计算平均预测概率 p̄(y=k)
    │         │
    │    最小化 H(p̄) → 1步梯度更新提示向量 p
    │
    └──→ 用更新后提示对x做最终预测 → 重置提示 → 下一个样本
```

### 3.4 TPT vs CoOp 关键区别

| 特性 | CoOp | TPT |
|------|------|-----|
| 训练数据 | 需要16-shot标注 | 不需要任何数据 |
| 优化时机 | 离线训练 | 在线测试时 |
| 提示共享 | 所有样本共用 | 每样本独立优化后重置 |
| 优化步数 | 数百步 | 仅1步 |
| 优化信号 | 交叉熵（有标签） | 预测熵（无标签） |

## 四、实验设置

### 4.1 数据集
- **自然分布偏移**：ImageNet → ImageNet-A（对抗样本）、ImageNet-V2、ImageNet-R（多风格）、ImageNet-Sketch
- **跨数据集泛化**：11个下游分类数据集

### 4.2 实现细节
- **骨干**：CLIP ViT-B/16, ResNet-50
- **提示长度 M**：4个token
- **增强数 N**：63
- **选择比例**：Top 10%低熵视图
- **优化步数**：1步
- **学习率**：5e-3 (AdamW)

## 五、实验结果

### 5.1 主要结果（CLIP ViT-B/16）

| 方法 | ImageNet | -A | -V2 | -R | -Sketch | OOD平均 |
|------|:-------:|:--:|:---:|:--:|:------:|:------:|
| CLIP零样本 | 66.73 | 47.87 | 60.83 | 73.98 | 46.09 | 57.19 |
| CLIP+80模板集成 | 68.34 | 49.89 | 61.88 | 77.65 | 48.24 | 59.42 |
| CoOp (16-shot) | 71.51 | 49.71 | 64.20 | 75.21 | 47.99 | 59.28 |
| CoCoOp (16-shot) | 71.02 | 50.63 | 64.07 | 76.18 | 48.75 | 59.91 |
| **TPT** | 68.98 | **54.77** | 63.45 | 77.06 | 47.94 | **60.81** |
| **TPT+CoOp** | 73.61 | **57.95** | **66.83** | 77.27 | 49.29 | **62.84** |

- ImageNet-A上提升**+6.9%**（最大提升数据集）
- 不需要标注数据，OOD平均优于需要16-shot的CoOp/CoCoOp
- TPT可与CoOp叠加使用

### 5.2 消融实验

**置信度选择比例**：

| 选择比例 | ImageNet-A |
|---------|:---------:|
| 无选择（100%） | 52.3 |
| Top 50% | 53.8 |
| **Top 10%** | **54.77** |
| Top 1% | 53.1 |

选择至关重要（+2.5%），Top 10%最佳。

**增强视图数量**：

| 视图数N | ImageNet-A |
|:------:|:---------:|
| 1 | 49.2 |
| 15 | 53.1 |
| **63** | **54.77** |
| 127 | 54.6 |

63之后收益递减，但每个视图需一次前向传播。

**优化步数**：

| 步数 | ImageNet-A |
|:----:|:---------:|
| 0（零样本） | 47.87 |
| **1** | **54.77** |
| 2 | 54.5 |
| 5 | 53.2 |

仅1步最佳，更多步过拟合。

## 六、局限性与不足

1. **推理速度慢约64倍**：每个样本需64次前向+1次反向传播，实时应用不可接受
2. **仅优化文本提示**：视觉层面的偏移无法通过文本端提示修正
3. **增强多样性有限**：随机裁剪+颜色抖动不能模拟所有偏移类型（→ DiffTPT的改进方向）
4. **单样本信号弱**：每样本独立适应，未利用测试流整体分布信息
5. **提示长度固定**：M=4是手动设定的
6. **每样本重置**：不累积跨样本知识
7. **ID提升有限**：原始ImageNet上几乎无提升，TPT主要解决OOD问题

## 七、后续影响

TPT开创了VLM测试时适应的全新方向，后续工作线：
- **DiffTPT**：扩散模型增强 → 更多样的视图
- **C-TPT**：校准约束 → 解决过度自信
- **RLCF**：CLIP奖励 → 替代熵目标
- **TDA/ZERO**：去掉反向传播 → 提速10倍+
- **StatA**：现实评估 → 暴露理想化设置问题
