## 一、研究背景

### 1.1 TPT中的偏差问题

TPT仅基于**单个无标签测试样本**的增强视图来优化提示。这个过程存在两种偏差：

**偏差1：增强偏差 (Augmentation Bias)**
- 测试时增强产生的视图可能不代表真实数据分布
- 激进增强（如严重裁剪）可能改变语义
- TPT的置信度选择只能部分缓解这个问题

**偏差2：优化偏差 (Optimization Bias)**
- 仅用一个样本的梯度信息做优化，方向可能有偏
- 单样本信号太弱，容易过拟合到该样本的特定噪声模式
- 梯度估计的方差极大

D2TPT的目标：同时消除（"双重去偏"）这两种偏差。

## 二、方法详解

### 2.1 模块一：动态检索增强调制 (DRAM)

**核心思想**：不要只依赖当前样本的增强视图，还要从一个**动态知识库**中检索相关的高置信度历史信息来辅助预测。

**知识库构建**：
- 维护一个动态知识库 $\mathcal{K} = \{(\mathbf{z}_i, \hat{y}_i, c_i)\}$
- 存储高置信度测试样本的：特征嵌入 $\mathbf{z}_i$、预测类别 $\hat{y}_i$、置信度 $c_i$
- FIFO策略维护，窗口大小为 $W$

**检索与调制**：
对当前测试样本 $x$：
1. 计算 $x$ 与知识库中每个样本的特征相似度
2. 检索Top-M个最相似的样本
3. 用检索样本的预测信息加权调制当前预测：

$$p_{DRAM}(y=k|x) = (1-\gamma) \cdot p_{model}(y=k|x) + \gamma \cdot \sum_{j \in \text{Top-M}} w_j \cdot \mathbf{1}[\hat{y}_j=k]$$

其中 $w_j$ 是相似度加权的归一化权重，$\gamma$ 是调制系数。

**去偏作用**：检索到的历史信息不依赖于当前样本的增强，提供了独立的分类信号，缓解增强偏差。

### 2.2 模块二：可靠性感知提示优化 (RAPO)

**核心思想**：在提示优化过程中引入两个约束来减少优化偏差。

**约束1：置信度加权集成**
不等权对待所有增强视图的预测，而是用置信度加权：

$$\bar{p}_{weighted}(y=k) = \frac{\sum_{i \in \mathcal{S}} c_i \cdot p(y=k|x^{(i)})}{\sum_{i \in \mathcal{S}} c_i}$$

高置信度视图对最终预测的贡献更大。

**约束2：跨模态一致性蒸馏**
利用CLIP的图文对齐知识作为正则化——优化后的提示应该与CLIP的零样本提示保持一定的语义一致性：

$$\mathcal{L}_{consist} = D_{KL}(p_{learned} \| p_{zeroshot})$$

防止学习到的提示偏离太远。

### 2.3 完整优化目标

$$\mathcal{L} = \underbrace{H(\bar{p}_{weighted})}_{\text{加权熵最小化}} + \alpha \underbrace{D_{KL}(p \| p_{zs})}_{\text{跨模态一致性}} + \beta \underbrace{\mathcal{L}_{DRAM}}_{\text{检索增强}}$$

### 2.4 流程图

```
测试样本 x
    │
    ├──→ 生成增强视图 → CLIP推理 → 置信度加权预测 p̄_weighted
    │
    ├──→ 动态知识库检索 → Top-M近邻 → 检索增强预测 p_DRAM
    │
    ├──→ CLIP零样本预测 p_zs（作为一致性锚点）
    │
    ├──→ 总损失 = H(p̄_weighted) + α·KL(p‖p_zs) + β·L_DRAM
    │         │
    │    反向传播 → 更新提示
    │
    ├──→ 最终预测：融合模型预测和DRAM预测
    │
    └──→ 如果高置信度 → 加入知识库
```

## 三、实验设置

- **骨干**：CLIP ViT-B/16, ViT-L/14
- **数据集**：ImageNet变体4个 + 11个下游分类数据集（共15个）
- **知识库窗口 $W$**：100
- **检索数 $M$**：5
- **基线**：TPT、C-TPT、DiffTPT、TDA、ZERO

## 四、实验结果

### 4.1 主要结果（CLIP ViT-B/16）

| 方法 | ImageNet-A | -V2 | -R | -Sketch | OOD平均 |
|------|:---------:|:---:|:--:|:------:|:------:|
| TPT | 54.77 | 63.45 | 77.06 | 47.94 | 60.81 |
| C-TPT | 55.1 | 63.5 | 77.1 | 48.0 | 60.93 |
| DiffTPT | 57.24 | 63.78 | 77.83 | 48.73 | 61.90 |
| **D2TPT** | **58.1** | **64.2** | **78.1** | **49.3** | **62.43** |

D2TPT在所有OOD数据集上取得最优，平均比TPT高+1.6%。

### 4.2 15个数据集平均

| 方法 | 15数据集平均Acc |
|------|:------------:|
| CLIP零样本 | 63.07 |
| TPT | 65.50 |
| **D2TPT** | **67.82** |

### 4.3 消融实验

| 组件 | ImageNet-A Acc |
|------|:---------:|
| TPT基线 | 54.77 |
| + DRAM（检索增强） | 56.8 (+2.0) |
| + RAPO（可靠性优化） | 56.2 (+1.4) |
| + DRAM + RAPO（完整D2TPT） | **58.1** (+3.3) |

两个模块贡献互补，结合效果最佳。

## 五、局限性与不足

1. **知识库的冷启动**：前几个样本没有历史信息可检索
2. **知识库污染**：错误的高置信度样本进入知识库会持续影响后续
3. **计算开销增加**：除了TPT的开销，还增加了检索和知识库管理的开销
4. **三个损失项的权重调节**：$\alpha$、$\beta$ 需要联合调优
5. **对极端域偏移的鲁棒性**：当域偏移太大时，CLIP零样本预测 $p_{zs}$ 质量差，一致性约束反而有害

## 六、核心价值

D2TPT是首个系统性分析和解决TPT中**双重偏差**问题的工作。其"检索增强+可靠性优化"的组合思路对后续VLM TTA方法有重要参考价值——**利用测试数据流的整体信息（而非仅单样本）来做适应**。
