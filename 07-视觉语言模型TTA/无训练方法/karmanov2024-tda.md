## 一、研究背景与动机

TPT系列方法每个样本都要做多步梯度优化，推理速度慢约10-64倍。在实际部署中，很多场景要求**近实时推理**（如自动驾驶、视频分析）。

TDA的核心问题：**能否在不做任何反向传播的情况下实现VLM的TTA？**

关键洞见：与其花计算资源优化提示参数，不如维护一个**动态缓存**——记住之前看到的高置信度样本的特征和预测，用这些缓存信息来辅助新样本的分类。这类似于人类的学习——我们会记住之前看到的可靠样例，并用它们来帮助识别新事物。

## 二、方法详解

### 2.1 双缓存机制

TDA维护两个缓存：

**正样本缓存 (Positive Cache)**：存储高置信度测试样本
- 条件：模型对该样本的预测熵 < 阈值 $\tau_{pos}$
- 存储内容：(特征嵌入, 预测类别, 置信度)
- 作用：为后续同类样本提供正面参考

**负样本缓存 (Negative Cache)**：存储低置信度测试样本
- 条件：模型对该样本的预测熵 > 阈值 $\tau_{neg}$
- 存储内容：(特征嵌入, 预测类别, 置信度)
- 作用：为后续样本提供反面参考（"不要像它那样分类"）

### 2.2 预测调制

当新样本 $x$ 到来时：

1. 计算 $x$ 的CLIP图像嵌入 $\mathbf{z} = f_I(x)$
2. 计算 $x$ 与正缓存中每个样本的余弦相似度 → 得到**正关联分数**
3. 计算 $x$ 与负缓存中每个样本的余弦相似度 → 得到**负关联分数**
4. 用正/负关联分数调制CLIP的原始预测logits：

$$\text{logit}_k^{final} = \text{logit}_k^{CLIP} + \alpha \sum_{j \in \text{pos}, y_j=k} \text{sim}(\mathbf{z}, \mathbf{z}_j) - \beta \sum_{j \in \text{neg}, y_j=k} \text{sim}(\mathbf{z}, \mathbf{z}_j)$$

直觉：如果新样本与正缓存中的某类样本相似 → 增加该类的logit；如果与负缓存中的某类样本相似 → 降低该类的logit。

### 2.3 动态缓存管理

- **容量限制**：每类最多存储 $K$ 个样本（防止内存无限增长）
- **更新策略**：当缓存满时，移除最旧的样本（FIFO策略）
- **自适应阈值**：$\tau_{pos}$ 和 $\tau_{neg}$ 可以根据测试数据流的整体熵分布自适应调整

### 2.4 流程图

```
测试样本 x
    │
    ├──→ CLIP图像编码器 → z (图像嵌入)
    │
    ├──→ CLIP零样本预测 → logit_k^CLIP
    │
    ├──→ 与正缓存计算相似度 → 正关联分数（"拉"向对应类）
    │
    ├──→ 与负缓存计算相似度 → 负关联分数（"推"离对应类）
    │
    ├──→ logit_k^final = logit_k^CLIP + α·正分数 - β·负分数
    │
    ├──→ 最终预测 y = argmax(logit^final)
    │
    └──→ 根据预测熵更新正/负缓存
         低熵 → 加入正缓存
         高熵 → 加入负缓存
```

## 三、实验设置

### 3.1 实现细节
- **骨干**：CLIP ViT-B/16
- **缓存大小**：每类K=3-5个样本
- **正阈值 $\tau_{pos}$**：使用均匀分布熵作为参考
- **调制系数**：$\alpha=1.0$, $\beta=0.5$
- **无反向传播，无参数更新**

### 3.2 数据集
- ImageNet及变体
- 11个下游分类数据集
- 细粒度分类数据集

## 四、实验结果

### 4.1 准确率对比（CLIP ViT-B/16）

| 方法 | ImageNet-A | -V2 | -R | -Sketch | OOD平均 | 推理速度 |
|------|:---------:|:---:|:--:|:------:|:------:|:-------:|
| CLIP零样本 | 47.87 | 60.83 | 73.98 | 46.09 | 57.19 | 1x |
| TPT | 54.77 | 63.45 | 77.06 | 47.94 | 60.81 | ~64x慢 |
| **TDA** | **54.2** | **63.1** | **77.3** | **48.1** | **60.68** | **~1x** |

关键发现：TDA在**几乎不损失准确率**的情况下，推理速度与原始CLIP相当（无反向传播）。

### 4.2 效率对比

| 方法 | 每样本推理时间(ms) | 每样本内存(MB) | 需要反向传播 |
|------|:----------------:|:------------:|:----------:|
| CLIP零样本 | 5 | 0 | ❌ |
| TPT | 320 | 1200 | ✅ |
| DiffTPT | 15000+ | 4000+ | ✅ |
| **TDA** | **8** | **<1** | **❌** |

TDA比TPT快约**40倍**，比DiffTPT快约**1875倍**。

## 五、局限性与不足

1. **冷启动问题**：刚开始时缓存为空，前几个样本无法获得缓存增强
2. **缓存污染**：如果早期预测大量错误，错误样本进入缓存会持续影响后续预测
3. **缓存大小敏感**：K太小信息不足，K太大引入噪声
4. **不修改模型**：适应能力上限受限于CLIP原始特征质量
5. **类别数影响**：类别数很多时，每类分配的缓存容量很小
6. **顺序依赖**：测试样本的到达顺序影响缓存内容，进而影响预测

## 六、核心创新总结

TDA的最大贡献是证明了**VLM的TTA不一定需要反向传播**——通过简单的缓存机制就能达到接近TPT的性能，同时保持与原始推理几乎相同的速度。这对于实际部署具有重大意义。
