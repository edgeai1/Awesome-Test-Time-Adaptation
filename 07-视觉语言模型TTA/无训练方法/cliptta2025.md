## 一、研究背景与动机

### 1.1 目标函数的根本性不匹配

CLIP是通过**对比学习**（InfoNCE损失）预训练的——拉近匹配的图文对，推远不匹配的。但TPT等TTA方法使用**熵最小化**作为测试时目标。

这存在一个根本性的不匹配：
- CLIP的训练目标关注**图文对之间的相对关系**（对比性）
- 熵最小化只关注**单个样本的预测分布形状**（自信度）

类比：如果你用足球训练了一个运动员，却用篮球规则测试他——虽然有些技能是通用的，但效果不会最好。

### 1.2 熵最小化导致的失败模式

作者系统分析了熵最小化在CLIP TTA中的两种失败模式：

**失败模式1：伪标签漂移 (Pseudo-label Drift)**
- 熵最小化逼模型对所有样本都给出自信预测
- 对困难/模糊样本，模型被逼着给出错误但自信的预测
- 这些错误的自信预测进入后续适应流程，污染模型

**失败模式2：类塌缩 (Class Collapse)**
- 持续的熵最小化使模型逐渐将越来越多样本归为少数几个"主导类"
- 小类样本被错误吸收到大类中
- 最终只剩下1-2个类别有预测

## 二、方法详解

### 2.1 软对比损失

CLIPTTA用**软对比损失**替代熵最小化，与CLIP的预训练目标对齐。

对测试样本 $x$ 及其增强视图 $\{x^{(1)}, ..., x^{(N)}\}$，定义软对比损失：

$$\mathcal{L}_{SC} = -\sum_{i=1}^N \sum_{k=1}^C \hat{q}_k^{(i)} \cdot \log \frac{\exp(\text{sim}(\mathbf{z}^{(i)}, \mathbf{t}_k) / \tau)}{\sum_{j=1}^C \exp(\text{sim}(\mathbf{z}^{(i)}, \mathbf{t}_j) / \tau)}$$

其中 $\hat{q}_k^{(i)}$ 是增强视图 $x^{(i)}$ 的**软伪标签**（使用EMA教师模型的预测），而非硬伪标签。

与标准对比损失的区别：
- 标准对比损失用二元标签（匹配/不匹配）
- 软对比损失用概率分布作为"软标签"
- 这避免了硬伪标签的量化噪声

### 2.2 为什么对比损失比熵最小化更好？

| 维度 | 熵最小化 | 软对比损失 |
|------|---------|----------|
| 优化目标 | $-\sum p \log p$（与CLIP无关） | $-\sum q \log p$（与CLIP一致） |
| 正则化效果 | 无（过度自信） | 有（软标签提供平滑） |
| 类间关系 | 忽略 | 通过对比保持 |
| 崩溃风险 | 高 | 低 |

### 2.3 离群对比曝光 (OCE) — 扩展到开放集

现实世界中测试数据可能包含训练时未见过的类别（OOD样本）。CLIPTTA通过**离群对比曝光**扩展到开放集场景：

$$\mathcal{L}_{OCE} = -\log(1 - \max_k p(y=k|x_{ood}))$$

对于被识别为OOD的样本，优化其对所有已知类的最大预测概率远离1，增强OOD检测能力。

OOD检测标准：如果一个样本对所有类别的最大相似度都低于阈值 $\delta$，则标记为OOD。

### 2.4 完整流程

```
测试样本 x
    │
    ├──→ 生成增强视图 {x¹, ..., xᴺ}
    │
    ├──→ EMA教师模型 → 软伪标签 q̂ₖ
    │
    ├──→ 软对比损失 L_SC（替代熵最小化）
    │
    ├──→ OOD检测 → 已知类? → 正常适应
    │                └── OOD? → OCE损失（推远所有类）
    │
    └──→ 总损失 = L_SC + λ·L_OCE → 更新参数
```

## 三、实验设置

- **骨干**：CLIP ViT-B/16, ViT-L/14
- **数据集**：75个数据集（！）——包括ImageNet变体、细粒度、域泛化、OOD检测基准
- **基线**：TPT、C-TPT、TDA、ZERO、Tent、CoTTA
- **评估模式**：episodic和online

## 四、实验结果

### 4.1 闭集分类（75个数据集）

| 方法 | 75数据集平均Acc |
|------|:------------:|
| CLIP零样本 | 61.3 |
| TPT | 63.8 |
| TDA | 64.1 |
| **CLIPTTA** | **66.2** |

在75个数据集的大规模评估中，CLIPTTA比TPT高+2.4%。

### 4.2 开放集OOD检测

| 方法 | FPR95↓ | AUROC↑ |
|------|:-----:|:-----:|
| CLIP零样本 | 45.2 | 85.3 |
| TPT | 52.1（更差） | 82.7（更差） |
| **CLIPTTA** | **38.5** | **89.1** |

关键发现：**TPT反而恶化了OOD检测性能**（FPR95从45.2%升到52.1%），因为熵最小化使模型对OOD样本也过度自信。CLIPTTA通过OCE机制显著改善了OOD检测。

### 4.3 长期在线适应

| 方法 | 10轮后性能变化 |
|------|:----------:|
| TPT | -8.3%（崩溃） |
| TDA | -2.1% |
| **CLIPTTA** | **-0.5%**（稳定） |

### 4.4 消融实验

| 组件 | 平均Acc | FPR95 |
|------|:------:|:-----:|
| TPT基线（熵最小化） | 63.8 | 52.1 |
| 替换为软对比损失 | 65.4 | 42.3 |
| + EMA教师 | 65.8 | 40.1 |
| + OCE | **66.2** | **38.5** |

每个组件都有明确贡献。

## 五、局限性与不足

1. **仍需反向传播**：计算开销与TPT相当
2. **EMA教师增加内存**：需要维护一份模型副本
3. **OCE的OOD阈值**：$\delta$ 需要预设，不同场景最优值不同
4. **软伪标签质量**：仍依赖模型初始预测质量——如果初始预测很差，软标签也不可靠
5. **对比损失需要负样本**：在类别数很少的任务上（如二分类），对比效果有限

## 六、核心价值

CLIPTTA的核心贡献是**统一训练和测试目标**这一思想：

> 既然CLIP用对比损失训练，那测试时也应该用对比损失适应——目标函数的一致性比复杂的方法设计更重要。

这一原则对整个TTA领域具有普遍指导意义：测试时适应目标应该与预训练目标保持一致。
