## 一、研究背景与动机

### 1.1 缓存方法的根本缺陷

TDA等基于缓存的方法维护一个固定大小的样本缓存，通过查找近邻来辅助分类。但这存在几个根本问题：

**问题1：容量有限**
缓存大小固定（如每类K=5个样本），只能记住最近的少数样本。当测试数据流很长时，大量有价值的历史信息被丢弃。

**问题2：丢弃导致遗忘**
当缓存满时，必须丢弃旧样本来给新样本腾空间。但旧样本可能属于不同的子域——丢弃它们等于遗忘了之前适应过的域。

**问题3：样本级别太粗糙**
缓存中的每个样本是一个独立的点，无法捕获测试数据的**整体分布特征**（如均值、方差、多模态性）。

### 1.2 DOTA的核心思想

> 不记忆单个样本，而是持续估计测试数据流的**底层分布**。

分布是对数据整体特征的紧凑表示——一个高斯分布只需要均值和协方差就能代表无数个样本。而且分布的更新是**增量式的**——新样本只需更新统计量，不需要丢弃旧样本。

## 二、方法详解

### 2.1 分布级建模

DOTA用**类条件分布**来建模测试数据：

$$p(\mathbf{z}|y=k) \approx \mathcal{N}(\boldsymbol{\mu}_k^{test}, \boldsymbol{\Sigma}_k^{test})$$

注意与ADAPT的区别：
- ADAPT使用**共享协方差**（所有类共用一个 $\Sigma$）
- DOTA使用**类特定协方差**（每类有自己的 $\Sigma_k$）

这使得DOTA能建模不同类别可能有不同的分布形状。

### 2.2 增量分布更新

用Welford在线算法增量更新均值和协方差：

$$\boldsymbol{\mu}_k^{(n)} = \boldsymbol{\mu}_k^{(n-1)} + \frac{1}{n}(\mathbf{z}^{(n)} - \boldsymbol{\mu}_k^{(n-1)})$$

$$\boldsymbol{S}_k^{(n)} = \boldsymbol{S}_k^{(n-1)} + (\mathbf{z}^{(n)} - \boldsymbol{\mu}_k^{(n-1)})(\mathbf{z}^{(n)} - \boldsymbol{\mu}_k^{(n)})^\top$$

$$\boldsymbol{\Sigma}_k^{(n)} = \frac{1}{n-1}\boldsymbol{S}_k^{(n)}$$

这种更新方式：
- **无需存储原始样本**：只需维护均值和累积矩阵
- **数值稳定**：Welford算法避免了大数减小数的精度问题
- **内存恒定**：无论处理多少样本，内存使用不变

### 2.3 贝叶斯预测

使用动态估计的分布计算后验：

$$p(y=k|\mathbf{z}) \propto p(\mathbf{z}|y=k) \cdot p(y=k)$$

其中先验 $p(y=k)$ 也通过观测频率动态估计。

### 2.4 双来源融合

最终预测融合两个信号：
1. **CLIP零样本预测**：$p_{CLIP}(y=k|\mathbf{z})$（来自文本-图像对齐）
2. **分布推断预测**：$p_{dist}(y=k|\mathbf{z})$（来自在线估计的高斯分布）

$$p_{final}(y=k|\mathbf{z}) = \alpha \cdot p_{CLIP}(y=k|\mathbf{z}) + (1-\alpha) \cdot p_{dist}(y=k|\mathbf{z})$$

$\alpha$ 随时间递减：初始阶段分布估计不准，更依赖CLIP；后期分布估计越来越准，逐渐主导。

### 2.5 与缓存方法的对比

| 维度 | TDA (缓存) | DOTA (分布) |
|------|-----------|-----------|
| 存储 | 存储K个样本 | 存储均值+协方差 |
| 内存 | 随K线性增长 | **恒定** |
| 丢弃旧信息 | 是（FIFO） | **否**（增量更新） |
| 信息利用 | 近邻查询（局部） | 分布推断（**全局**） |
| 遗忘问题 | 严重 | **无** |
| 更新复杂度 | O(K) 查找 | O(d²) 协方差更新 |

### 2.6 流程图

```
测试样本流 x₁, x₂, ..., xₜ
    │
    ├──→ CLIP编码 → zₜ
    │
    ├──→ CLIP零样本预测 → p_CLIP(y|zₜ)
    │
    ├──→ 分布推断预测 → p_dist(y|zₜ)
    │         │
    │    基于当前估计的类条件高斯分布
    │    {N(μ₁ᵗ,Σ₁ᵗ), ..., N(μᶜᵗ,Σᶜᵗ)}
    │
    ├──→ 融合预测 p_final = α·p_CLIP + (1-α)·p_dist
    │
    ├──→ 预测 ŷₜ
    │
    └──→ 如果高置信度：增量更新 μ_ŷ 和 Σ_ŷ
         （Welford算法，内存恒定）
```

## 三、实验结果

### 3.1 长序列在线适应（核心实验）

| 方法 | 1K样本 | 5K样本 | 10K样本 | 50K样本 | 趋势 |
|------|:-----:|:-----:|:------:|:------:|:---:|
| TDA | 54.0 | 53.2 | 52.1 | 49.8 | ↓ 持续退化 |
| BCA | 55.2 | 55.0 | 55.0 | 54.5 | → 基本稳定 |
| **DOTA** | **55.5** | **56.1** | **56.5** | **57.2** | **↑ 持续提升** |

关键发现：
- TDA因丢弃旧样本持续退化
- BCA相对稳定但不提升
- **DOTA是唯一持续提升的方法**——因为更多数据=更准确的分布估计

### 3.2 内存使用（50K样本后）

| 方法 | 内存使用 |
|------|:------:|
| TDA (K=5) | 恒定（小） |
| TDA (K=50) | 恒定（中） |
| 无限缓存（理论上限） | 线性增长 → **溢出** |
| **DOTA** | **恒定（小）** |

DOTA以恒定内存实现了"无限缓存"的效果。

### 3.3 域切换场景

| 方法 | 域A→域B→域A | 域A最终Acc |
|------|:--------:|:--------:|
| TDA | 域A信息在域B期间被丢弃 | 48.2（遗忘） |
| **DOTA** | 域A分布保持在均值/协方差中 | **54.1**（保持） |

DOTA在域切换后能**保持之前域的知识**，因为分布统计量不会被后续数据完全覆盖。

### 3.4 消融实验

| 设置 | 10K样本后Acc |
|------|:--------:|
| 仅CLIP零样本 | 47.87 |
| + 类均值估计（共享Σ=I） | 54.8 |
| + 类特定Σ估计 | 55.9 |
| + 动态融合系数α | 56.3 |
| + 先验频率估计 | **56.5** |

类特定协方差比共享协方差好+1.1%。

## 四、局限性与不足

1. **高斯假设的局限**：复杂分布（多模态、重尾）用高斯近似可能不够
2. **类特定协方差的计算量**：每类维护一个 $d \times d$ 矩阵（$d$=768），C个类需要 $C \times d^2$ 空间。当类别数和特征维度都很大时有压力
3. **协方差矩阵的逆**：分类时需要计算 $\Sigma_k^{-1}$，对于高维矩阵不稳定（可能奇异）
4. **初期估计不准**：前几十个样本时，分布估计的方差很大
5. **伪标签质量影响**：错误归类的样本会污染对应类的分布估计
6. **非平稳分布**：如果测试数据的分布本身在变化（如持续域偏移），历史统计量可能过时

## 五、核心价值

DOTA的核心贡献是**从样本级记忆升级为分布级记忆**：

> 人类学习不是记住每个见过的例子，而是形成对每个概念的"分布性理解"。DOTA将这一认知原理应用于TTA。

这种范式相比缓存方法有三个根本优势：
1. **信息不丢失**：每个样本的信息都融入了分布统计量
2. **内存恒定**：无论处理多少样本，存储不变
3. **越多越好**：遵循大数定律，估计随样本增多而收敛
