# 🔄 持续/在线测试时适应 (Continual / Online TTA)

> 处理持续变化的分布偏移，需要在不遗忘的前提下不断适应新域。

---

## 1. CoTTA — Continual Test-Time Domain Adaptation

| 项目 | 内容 |
|------|------|
| **作者** | Qin Wang, Olga Fink, Luc Van Gool, Dengxin Dai |
| **发表** | CVPR 2022 |
| **链接** | [arXiv 2203.13591](https://arxiv.org/abs/2203.13591) |

### 研究背景与动机
Tent假设测试数据来自单一域偏移，但现实中偏移是**持续变化**的（如自动驾驶中天气从晴到雨再到雪）。在持续偏移下，Tent会逐渐崩溃——错误的伪标签积累导致模型越来越偏，最终完全失效。CoTTA首次系统研究了**持续测试时适应**（Continual TTA）问题。

### 核心贡献
1. 首次定义**持续TTA**的问题设置：模型需要依次适应一系列变化的域
2. 提出**加权增强伪标签**：对同一样本的多个增强视图的预测做加权平均，减少伪标签噪声
3. 提出**随机权重恢复**（Stochastic Restore）：以小概率将部分参数恢复到源模型值，防止长期漂移

### 方法详解
- **增强伪标签**：对测试样本 $x$ 生成 $N$ 个增强视图，计算平均预测作为伪标签，比单一预测更可靠
- **随机恢复**：每次更新后，以概率 $p$ 将每个参数恢复到其源模型初始值。这提供了一种"软重置"机制——模型不会偏离源模型太远，同时保留了适应能力
- **教师模型**（EMA）：用指数移动平均维护教师模型，教师提供更稳定的伪标签

### 实验设置与结果
- **数据集**：CIFAR-10/100-C（持续设置：15种损坏依次出现）、ImageNet-C
- **评估设置**：模型从第1种损坏适应到第15种，不重置
- **结果**：Tent在第15种损坏时准确率比不适应还低10%（崩溃），CoTTA始终保持正向提升
- 在ImageNet-C持续设置上比Tent平均高约8%

### 局限性
1. 随机恢复是一种"保守"策略——在快速变化的域中可能适应不够快
2. 恢复概率 $p$ 是固定的，不能根据偏移程度自适应调整
3. 增强伪标签增加约 $N$ 倍推理开销
4. 在域偏移非常大时，伪标签质量仍然不够好

## 2. NOTE — Robust Continual Test-time Adaptation Against Temporal Correlation

| 项目 | 内容 |
|------|------|
| **作者** | Taesik Gong, Jongheon Jeong, Taewon Kim, Yewon Kim, Jinwoo Shin, Sung-Ju Lee |
| **发表** | NeurIPS 2022 |
| **链接** | [GitHub](https://github.com/TaesikGong/NOTE) |

### 研究背景与动机
持续TTA的现有方法（包括CoTTA）都假设测试数据是**独立同分布**（i.i.d.）到达的，但实际中测试数据往往有**时序相关性**。例如自动驾驶中连续几帧都是同一场景、同一类物体。当一个batch中都是同一类样本时，BN统计量严重偏向该类，导致TTA崩溃。

### 核心贡献
1. 首次识别并解决**时序相关**（temporally correlated）测试数据流的TTA问题
2. 提出**实例感知BN（IABN）**：为每个样本独立计算归一化统计量，消除批内样本的相互影响
3. 提出**预测平衡Reservoir采样**：维护一个类别平衡的memory buffer

### 方法详解
- **IABN**：标准BN用batch统计量归一化所有样本，IABN对每个样本独立计算均值/方差进行归一化（类似Instance Norm），但保留BN的仿射参数
- **Reservoir采样**：维护一个固定大小的memory buffer，用Reservoir采样策略保持类别平衡。适应时从buffer中采样而非使用当前batch
- 两者结合：IABN解决单样本归一化问题，Reservoir解决类别不平衡问题

### 实验设置与结果
- **数据集**：CIFAR-10/100-C、ImageNet-C（时序相关设置：同一类的样本连续出现）
- **结果**：在时序相关设置下，Tent准确率降低20%以上，NOTE保持稳定
- 在i.i.d.设置下也与Tent相当（不损失性能）

### 局限性
1. IABN增加了约30%的计算开销
2. Memory buffer需要额外存储空间
3. 类别平衡采样需要估计每个样本的类别，依赖于模型预测质量

## 3. RoTTA — Robust Test-Time Adaptation in Dynamic Scenarios

| 项目 | 内容 |
|------|------|
| **作者** | Longhui Yuan, Binhui Xie, Shuang Li |
| **发表** | CVPR 2023 |
| **链接** | [arXiv 2303.13899](https://arxiv.org/abs/2303.13899) |

### 研究背景与动机
现实中的持续TTA面临三重挑战：(1) BN统计量不稳定；(2) 类别分布不均衡；(3) 测试数据的时效性（新到的数据更能反映当前域）。RoTTA综合解决这三个问题。

### 核心贡献
1. 提出**鲁棒BN（Robust BN）**：修正BN统计量的估计
2. 提出**类别平衡采样**（Category-Balanced Sampling）：memory bank中保持类别平衡
3. 提出**时间感知重加权**（Timeliness-Aware Reweighting）：近期样本权重更高
4. 使用师生模型框架增强稳定性

### 方法详解
- **鲁棒BN**：用全局EMA统计量替代batch统计量，消除batch大小的影响
- **类别平衡Memory**：为每个类别维护固定数量的代表样本，用伪标签分类入库
- **时间重加权**：样本的权重随时间衰减，更早到的样本可能来自旧域，权重降低
- **师生框架**：教师通过EMA更新，学生在教师监督下训练

### 实验设置与结果
- **数据集**：CIFAR-10/100-C、ImageNet-C（持续+类不平衡+时序相关）
- **结果**：在三种挑战同时存在的最难设置下，比CoTTA高5%，比NOTE高3%

### 局限性
1. 三个组件各有超参数，整体调参复杂度高
2. Memory bank的大小需要平衡存储和效果
3. 时间衰减策略假设域是线性变化的，对突变域偏移可能不适用

## 4. SoTTA — Robust Test-Time Adaptation on Noisy Data Streams

| 项目 | 内容 |
|------|------|
| **作者** | Taesik Gong et al. |
| **发表** | NeurIPS 2023 |
| **链接** | [arXiv 2310.10074](https://arxiv.org/abs/2310.10074) |

### 研究背景与动机
NOTE团队的后续工作。除了时序相关性，现实中的测试数据流还可能包含**噪声样本**——不属于任何已知类的样本、严重损坏的样本等。这些噪声样本会严重干扰TTA的适应过程。

### 核心贡献
1. 首次研究**噪声数据流**下的TTA问题
2. 提出**高置信度均匀类采样**（High-Confidence Uniform-Class Sampling）：同时过滤噪声和保持类别平衡
3. 提出**熵-锐度最小化**（Entropy-Sharpness Minimization）：在平坦区域优化以增强对噪声的鲁棒性

### 方法详解
- **噪声过滤**：基于预测置信度过滤可能的噪声样本，仅保留高置信度样本
- **均匀类采样**：在高置信度样本中保持类别均匀分布
- **熵-锐度联合优化**：不仅最小化熵，还最小化熵关于参数的锐度（损失面的曲率），使模型更不容易被噪声样本"推离"

### 实验设置与结果
- **数据集**：CIFAR-10/100-C + 注入的噪声（高斯噪声、OOD样本）
- **噪声比例**：10%-50%
- **结果**：在50%噪声比例下，Tent性能崩溃至随机猜测水平，SoTTA仍保持约70%准确率

### 局限性
1. 置信度阈值的设定对性能影响大
2. 在高噪声比例下，可用样本太少，适应信号不足
3. 锐度最小化增加约2倍计算成本

## 5. EcoTTA — Memory-Efficient Continual TTA via Self-Distilled Regularization

| 项目 | 内容 |
|------|------|
| **作者** | Junha Song et al. |
| **发表** | CVPR 2023 |
| **链接** | [arXiv 2303.01904](https://arxiv.org/abs/2303.01904) |

### 研究背景与动机
持续TTA需要长期运行，而边缘设备（手机、嵌入式系统）的内存有限。现有方法（CoTTA、RoTTA）需要存储教师模型、memory bank等，内存开销太大。EcoTTA的目标是在**极低内存**下实现有效的持续TTA。

### 核心贡献
1. 提出**轻量级元网络**：在冻结的原始网络上附加极小的可训练模块
2. 提出**自蒸馏正则化**：用冻结的原始模型输出作为正则化目标，防止遗忘
3. 内存占用仅为CoTTA的约1/3

### 方法详解
- **冻结原网络**：保持预训练模型完全冻结（连BN都不更新）
- **元网络**：在每个残差块后添加一个很小的适配网络（如1×1卷积），仅训练这些适配器
- **自蒸馏**：适配后的输出与冻结模型的输出之间做KL散度，防止偏离太远

### 实验设置与结果
- **数据集**：CIFAR-10/100-C、ImageNet-C
- **内存对比**：EcoTTA需约0.3M参数更新（CoTTA需约1.2M）
- **结果**：准确率与CoTTA接近，但内存和计算效率高3-4倍

### 局限性
1. 适配器的设计需要针对不同架构调整
2. 冻结原网络可能过于保守，限制了适应能力
3. 在大偏移场景下，小适配器的适应能力可能不够

## 6. PeTTA — Persistent Test-time Adaptation in Recurring Testing Scenarios

| 项目 | 内容 |
|------|------|
| **作者** | Trung-Hieu Hoang et al. |
| **发表** | NeurIPS 2024 |
| **链接** | [arXiv 2311.18193](https://arxiv.org/abs/2311.18193) |

### 研究背景与动机
现实世界中域偏移可能**重复出现**（如自动驾驶中白天→夜晚→白天循环）。现有持续TTA方法没有考虑这种循环模式，可能在遇到之前适应过的域时仍然从头适应，浪费了之前的经验。

### 核心贡献
1. 首次定义**重复/循环TTA**（Recurring TTA）场景
2. 提出检测模型向崩溃发散的机制，并动态调整适应策略
3. 实现了真正的"终身"TTA——模型能无限期运行而不崩溃

### 方法详解
- **崩溃检测**：监控模型预测的熵分布。当熵分布偏离正常范围时，判定模型在崩溃
- **动态策略切换**：正常状态→正常适应；崩溃趋势→降低学习率或暂停适应
- **经验库**：保存成功适应过的域的模型快照，遇到相似域时快速恢复

### 实验设置与结果
- **数据集**：CIFAR-10/100-C（循环10轮）、ImageNet-C（循环5轮）
- **结果**：在10轮循环后，Tent/CoTTA准确率持续下降，PeTTA保持稳定

### 局限性
1. 崩溃检测的阈值需要调整
2. 经验库增加存储开销
3. 域相似度的度量是启发式的

## 7. RDBO — Reshaping the Online Data Buffering and Organizing for Continual TTA

| 项目 | 内容 |
|------|------|
| **作者** | Li et al. |
| **发表** | ECCV 2024 |
| **链接** | [arXiv 2407.09367](https://arxiv.org/abs/2407.09367) |

### 研究背景与动机
持续TTA中的memory buffer策略对性能影响很大，但现有方法（如RoTTA的类别平衡采样）过于简单。RDBO深入研究了"如何更好地组织和利用缓存数据"。

### 核心贡献
1. 提出**不确定性感知缓存**：根据样本的不确定性决定是否入库
2. 提出**基于图的类关系保持**：在缓存中保持类间关系结构
3. 提出**伪目标重放**：用缓存样本构建更可靠的适应信号

### 实验设置与结果
- 在CIFAR-10/100-C持续设置下比RoTTA提升约2-3%

### 局限性
1. 图构建增加计算开销
2. 缓存策略的多个超参数需要联合调整

## 8. PALM — Pushing Adaptive Learning Rate Mechanisms for Continual TTA

| 项目 | 内容 |
|------|------|
| **作者** | Maharana et al. |
| **发表** | AAAI 2025 |
| **链接** | [arXiv 2403.10650](https://arxiv.org/abs/2403.10650) |

### 研究背景与动机
TTA中学习率的选择至关重要——太大导致崩溃，太小适应不够。PALM提出用梯度信息自动调整每层的学习率。

### 核心贡献
1. 提出**层级自适应学习率**：不同层用不同学习率
2. 用KL散度到均匀分布的梯度幅度来选择学习率（**无需伪标签**）
3. 在持续场景下比固定学习率方法显著更稳定

### 方法详解
- 对每层计算KL散度关于参数的梯度范数
- 梯度范数大的层→偏移大→学习率高
- 梯度范数小的层→偏移小→学习率低
- 完全不使用伪标签，避免了伪标签噪声

### 实验设置与结果
- **数据集**：CIFAR-10/100-C、ImageNet-C持续设置
- **结果**：比固定学习率的Tent稳定50%以上

### 局限性
1. 梯度幅度与最优学习率的映射关系是启发式的
2. 层级学习率增加了一定的计算开销

## 持续TTA的核心挑战总结

```
┌─────────────────────────────────────────────────────────────┐
│                  持续TTA核心挑战与解决方案                      │
├──────────────┬────────────────┬──────────────────────────────┤
│   灾难性遗忘   │    错误累积     │    现实场景复杂性              │
│  (Forgetting)  │ (Error Accum.) │  (Real-world Complexity)     │
├──────────────┼────────────────┼──────────────────────────────┤
│ CoTTA(随机恢复)│ SoTTA(噪声过滤) │ NOTE(时序相关)               │
│ EcoTTA(冻结网络)│ RoTTA(师生+时间) │ RoTTA(类不平衡)            │
│ PeTTA(崩溃检测)│ RDBO(智能缓存)  │ PeTTA(循环域偏移)            │
│ PALM(自适应LR) │               │ PALM(层级学习率)              │
└──────────────┴────────────────┴──────────────────────────────┘
```
