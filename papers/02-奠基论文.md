## 1. TTT — Test-Time Training with Self-Supervision for Generalization under Distribution Shifts

| 项目 | 内容 |
|------|------|
| **作者** | Yu Sun, Xiaolong Wang, Zhuang Liu, John Miller, Alexei A. Efros, Moritz Hardt |
| **发表** | ICML 2020 |
| **链接** | [arXiv 1909.13231](https://arxiv.org/abs/1909.13231) |

### 研究背景
深度学习模型在训练分布和测试分布一致的情况下表现优异，但在面临分布偏移（Distribution Shift）时性能急剧下降。传统方法要么依赖大量目标域有标签数据（域适应），要么试图训练一个在所有域上都泛化的模型（域泛化），都有各自的局限。

### 动机 (Motivation)
能否在测试时利用每个测试样本本身的信息来"临时"调整模型？关键挑战是测试时没有标签。作者的核心洞见是：虽然主任务（如分类）的标签不可得，但**自监督任务不需要标签**——可以设计一个与主任务共享表征的辅助自监督任务，在测试时通过该任务的梯度更新来适应分布偏移。

### 核心贡献
1. 首次提出**测试时训练（Test-Time Training, TTT）**的范式
2. 设计了一个共享编码器的Y形网络结构：主任务分支（分类）+ 辅助任务分支（旋转预测）
3. 在测试时，对每个测试样本通过自监督损失更新共享编码器
4. 开创了TTA这一全新研究方向

### 方法详解
- **训练阶段**：同时训练主任务损失 $L_{main}$ 和自监督损失 $L_{ssl}$（预测图像旋转角度0°/90°/180°/270°），两个任务共享特征提取器
- **测试阶段**：对每个测试样本 $x_{test}$，先通过自监督损失 $L_{ssl}(x_{test})$ 计算梯度，更新共享特征提取器的参数，然后再用更新后的模型做主任务预测
- 每个测试样本独立适应，适应后参数重置回原始模型

### 实验设置
- **数据集**：CIFAR-10-C（15种损坏类型 × 5级强度）、自构建的视频姿态估计偏移数据
- **骨干网络**：ResNet-26
- **基线对比**：标准训练（ERM）、数据增强、域泛化方法
- **评估指标**：损坏数据上的分类准确率

### 实验结果
- 在CIFAR-10-C上，TTT将平均错误率从43.5%降低到36.6%（Level 5损坏），相对提升约16%
- 在视频姿态估计任务上同样有效
- 损坏越严重，TTT的提升越明显

### 局限性与不足
1. **需要修改训练流程**：必须在训练阶段引入辅助任务，无法直接应用于已有的预训练模型
2. **辅助任务设计依赖先验知识**：旋转预测这一辅助任务对自然图像有效，但不一定适用于所有场景（如医学影像、遥感图像）
3. **辅助任务与主任务的对齐性**：无法保证自监督任务的梯度方向一定有利于主任务
4. **计算开销**：每个测试样本都需要梯度计算和参数更新，增加推理时延
5. **仅在单样本水平适应**，没有利用测试数据流的整体分布信息

## 2. SHOT — Do We Really Need to Access the Source Data? Source Hypothesis Transfer for Unsupervised Domain Adaptation

| 项目 | 内容 |
|------|------|
| **作者** | Jian Liang, Dapeng Hu, Jiashi Feng |
| **发表** | ICML 2020 |
| **链接** | [arXiv 2002.08546](https://arxiv.org/abs/2002.08546) |

### 研究背景
无监督域适应（UDA）的经典方法（如DANN、CDAN）都假设可以同时访问源域和目标域数据。然而在实际部署中，源域数据可能由于隐私保护、存储限制或知识产权问题而无法获取。

### 动机 (Motivation)
在不访问源域数据的情况下，仅利用预训练的源模型和无标签的目标域数据，能否实现有效的域适应？这就是**无源域适应（Source-Free Domain Adaptation, SFDA）**问题。SHOT的核心假设是：源域训练好的分类器头（即"源假设"，Source Hypothesis）已经编码了判别性的类别边界信息，只要将目标域的特征对齐到源分类器期望的分布上，就可以实现良好的适应。

### 核心贡献
1. 首次系统提出**无源域适应**的问题设置和解决方案
2. 提出冻结源分类器、仅调整特征提取器的适应策略
3. 结合信息最大化（互信息最大化 + 多样性正则化）和伪标签自训练实现适应
4. 在多个域适应基准上超越需要源数据的传统UDA方法

### 方法详解
- **冻结分类器**：保持源域训练好的全连接分类层不变
- **信息最大化**：最大化模型输出与输入之间的互信息，包括两个子目标：(1) 个体熵最小化——使每个样本的预测更自信；(2) 全局多样性——确保整体预测分布均匀，避免所有样本崩溃到同一类
- **伪标签自训练**：使用高置信度的模型预测作为伪标签，进一步微调特征提取器
- 两步交替优化：先做信息最大化预热，再引入伪标签精调

### 实验设置
- **数据集**：Office-31、Office-Home、VisDA-C、DomainNet
- **骨干网络**：ResNet-50/101
- **基线对比**：DANN、CDAN、MCD等传统UDA方法（需要源数据）
- **评估指标**：分类准确率

### 实验结果
- Office-31上平均准确率达90.1%，超越多数需要源数据的UDA方法
- VisDA-C上达82.9%，比最佳UDA方法高2%+
- 关键发现：不需要源数据也能做到甚至超越需要源数据的方法

### 局限性与不足
1. **假设分类器质量**：依赖源域分类器的质量，如果源域训练不充分，适应效果受限
2. **需要目标域足够多数据**：信息最大化和伪标签都需要一定量的目标数据，不适用于单样本/小批次场景
3. **离线适应**：属于适应阶段的方法，不适用于纯在线TTA场景
4. **伪标签噪声**：伪标签质量在域偏移大时显著下降，导致错误积累
5. **未考虑持续偏移**：只处理单一域偏移，不能处理持续变化的域

## 3. AdaptiveBN — Improving Robustness against Common Corruptions by Covariate Shift Adaptation

| 项目 | 内容 |
|------|------|
| **作者** | Steffen Schneider, Evgenia Rusak, Luisa Eck, Oliver Bringmann, Wieland Brendel, Matthias Bethge |
| **发表** | NeurIPS 2020 |
| **链接** | [GitHub](https://github.com/bethgelab/robustness) |

### 研究背景
深度神经网络中的批归一化（BN）层在训练时会计算并存储训练数据的均值和方差统计量。推理时，这些固定的统计量被用来归一化特征。当测试数据的分布与训练数据不同时，这些统计量就会导致特征归一化不准确。

### 动机 (Motivation)
BN层的统计量本质上是对数据分布的"快照"。如果测试数据的分布偏移了，那用训练分布的快照来归一化就是不合理的。最简单的想法就是：**用测试数据自己的统计量来替换训练时的统计量**。

### 核心贡献
1. 揭示了BN统计量错配是模型对分布偏移敏感的关键原因
2. 提出了极简方案：在测试时用目标域数据重新计算BN的均值和方差
3. 证明仅修改BN统计量（**无需任何梯度计算或参数更新**）就能显著提升鲁棒性
4. 为后续所有基于BN的TTA方法提供了理论基础

### 方法详解
- **核心操作**：在测试时，将模型设为`train`模式（PyTorch中的`model.train()`），让BN层重新计算当前批次测试数据的均值和方差，替代训练时存储的统计量
- 不修改BN的可学习参数（γ和β），也不修改网络的其他参数
- 可以使用单一批次的统计量，也可以用多个批次的指数移动平均

### 实验设置
- **数据集**：ImageNet-C（15种损坏 × 5级强度）
- **骨干网络**：多种ResNet变体、EfficientNet等
- **基线对比**：标准推理、对抗训练、数据增强等鲁棒性方法

### 实验结果
- 在ImageNet-C上平均减少约20-40%的错误率，且完全不需要训练
- 与对抗训练等昂贵方法相当甚至更优
- 方法的计算开销几乎为零

### 局限性与不足
1. **需要足够大的批次**：BN统计量的准确估计需要足够多的样本（通常≥32），小批次或单样本场景下估计不准
2. **批次内类别分布敏感**：如果批次中类别分布严重不均，BN统计量会被偏向主导类
3. **不适应BN的仿射参数**：只修改统计量，不更新γ和β，适应能力有限
4. **不适用于无BN的网络**：如LayerNorm/GroupNorm的ViT等架构
5. **需要模型包含BN层**：不是通用的适应策略

## 4. Tent — Fully Test-Time Adaptation by Entropy Minimization

| 项目 | 内容 |
|------|------|
| **作者** | Dequan Wang, Evan Shelhamer, Shaoteng Liu, Bruno Olshausen, Trevor Darrell |
| **发表** | ICLR 2021 |
| **链接** | [arXiv 2006.10726](https://arxiv.org/abs/2006.10726) |

### 研究背景
TTT需要修改训练流程加入辅助任务，AdaptiveBN只能调整统计量而不能更新参数，这两种方法都有明显的局限。领域急需一个既不需要修改训练流程，又能在测试时通过梯度更新实现更强适应的方法。

### 动机 (Motivation)
Tent的动机非常直觉：如果一个模型对某个测试样本的预测非常不确定（高熵），说明模型遇到了困难——这很可能是分布偏移导致的。那么，通过**最小化预测熵**来更新模型参数，就可以使模型变得更加自信，从而适应偏移。关键约束是只更新BN层的仿射参数（γ和β），这比更新全部参数更稳定且高效。

### 核心贡献
1. **定义了完全测试时适应（Fully TTA）**的标准范式：不需要修改训练过程、不需要源数据、仅用未标注测试数据
2. 提出**熵最小化**作为TTA的核心优化目标
3. 仅更新BN层的仿射参数（γ, β），其余参数冻结
4. 方法极其简洁（核心代码约10行），成为几乎所有后续TTA工作的基线

### 方法详解
- **优化目标**：最小化模型输出的Shannon熵 $H(y) = -\sum_c p(y=c|x) \log p(y=c|x)$
- **可更新参数**：仅BN层的仿射参数 γ（缩放）和 β（偏移），约占模型总参数的<1%
- **BN统计量**：同时用测试批次的统计量替换训练统计量（继承AdaptiveBN思想）
- **在线更新**：每个批次计算一次前向传播、一次反向传播，更新参数后处理下一批次
- 可以选择每个样本适应后重置参数（episodic模式），也可以持续适应不重置（online模式）

### 实验设置
- **数据集**：CIFAR-10-C、CIFAR-100-C、ImageNet-C
- **骨干网络**：WideResNet-28、ResNeXt-29、ResNet-50
- **基线对比**：标准推理、AdaptiveBN、TTT、PredBN
- **评估指标**：损坏数据上的错误率
- **适应模式**：在线（online）和分片（episodic）

### 实验结果
- CIFAR-10-C (Level 5)：错误率从43.5%降至18.6%，相比AdaptiveBN进一步降低了约40%
- CIFAR-100-C (Level 5)：错误率从46.4%降至29.1%
- ImageNet-C：错误率从76.7%降至62.0%（ResNet-50）
- 相比TTT：性能接近，但**不需要修改训练流程**

### 局限性与不足
1. **小批次退化**：当批次大小很小（如1-4）时，BN统计量估计不准确，性能显著下降
2. **长期适应不稳定**：在持续偏移的在线场景中，持续更新会导致模型逐渐崩溃
3. **熵最小化的模式崩溃**：当域偏移很大时，熵最小化可能导致所有样本被分到同一类（模式崩溃）
4. **无样本选择机制**：对所有样本一视同仁地做熵最小化，低质量样本（如离群值）可能引入噪声梯度
5. **超参数敏感**：学习率的选择对性能影响很大，且不同偏移类型的最优学习率不同
6. **仅适用于包含BN层的网络**：不能直接应用于ViT等使用LayerNorm的架构

## 5. ARM — Adaptive Risk Minimization: Learning to Adapt to Domain Shift

| 项目 | 内容 |
|------|------|
| **作者** | Marvin Zhang, Henrik Marklund, Nikita Dhawan, Abhishek Gupta, Sergey Levine, Chelsea Finn |
| **发表** | NeurIPS 2021 |
| **链接** | [arXiv 2007.02931](https://arxiv.org/abs/2007.02931) |

### 研究背景
传统ERM训练的模型在测试时是固定的，无法根据测试数据的特点进行调整。而TTT/Tent等方法虽然在测试时做适应，但适应策略（如损失函数、学习率）是手动设计的，不一定是最优的。

### 动机 (Motivation)
能否让模型**在训练阶段就学习如何在测试时进行适应**？类似于元学习中MAML的思想——训练一个"擅长快速适应"的模型初始化，而不是一个"在所有域上都能用"的固定模型。ARM的洞见是：与其在训练时追求所有域上的低平均风险（ERM），不如训练一个在面对新域时能快速降低风险的模型（Adaptive Risk Minimization）。

### 核心贡献
1. 提出将测试时适应能力纳入训练目标的**自适应风险最小化**框架
2. 设计了基于上下文网络（context network）的适应机制，用小批次测试样本的聚合信息指导适应
3. 不需要手动设计适应策略，适应能力是"学出来"的

### 方法详解
- **训练框架**：模拟MAML式的双层优化——外层优化模型初始化，内层模拟测试时适应过程
- **上下文网络**：接收一个小批次的测试样本，生成上下文向量，用该向量调制模型的特征
- **训练过程**：将训练域分成不同子域，每个子域模拟一个"测试时适应"的episode

### 实验设置
- **数据集**：CIFAR-10-C、FMoW（卫星图像时序域偏移）、Camelyon17（医学组织病理学）
- **基线对比**：ERM、DRO（分布鲁棒优化）、MAML、Tent等

### 实验结果
- 在FMoW和Camelyon17上超越ERM和域泛化方法
- 在分布偏移幅度大的场景中优势更明显
- 学出的适应策略比手动设计的Tent策略更有效

### 局限性与不足
1. **训练成本高**：元学习式的双层优化显著增加训练时间和内存
2. **需要训练时已知域划分**：需要多个训练域来模拟测试时的适应过程
3. **上下文网络泛化性**：学出的适应策略可能过拟合于训练时见过的偏移类型
4. **不适用于已有预训练模型**：需要从头用ARM框架训练，不能后处理地应用于现有模型
5. **实验规模有限**：未在ImageNet规模上验证

## 发展脉络

```
2020 ── TTT (自监督辅助任务)   ← 开创"测试时训练"概念
     ── SHOT (冻结分类器+信息最大化) ← 开创"无源域适应"
     ── AdaptiveBN (替换BN统计量)  ← 最简TTA基线

2021 ── Tent (熵最小化+BN仿射参数) ← 🔥 定义"完全TTA"标准范式
     ── ARM (元学习适应能力)      ← "学会适应"的思路

     所有后续TTA方法基本都是在解决Tent的局限性：
     ├── 小批次问题 → MEMO, LAME
     ├── 长期不稳定 → CoTTA, RoTTA
     ├── 样本选择 → EATA, SAR, DeYO
     ├── 模式崩溃 → DELTA, COME
     └── 非BN架构 → 提示调优系列
```
